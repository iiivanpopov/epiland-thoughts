## Проблема бізнесу

> **Можливо**. насправді ми поки що цього не знаємо.

Зараз менеджери перевантажені однотипними питаннями про ціни, графік роботи та локації парків. Через це виникають наступні бізнес-проблеми:

**Рутина замість продажів:** Менеджери витрачають більшість часу на базові відповіді. Через це у них не залишається часу та ресурсу на якісний продаж дорогих пакетів (наприклад, днів народжень).

**Черга:** Коли сім'я хоче швидко дізнатися деталі про святкування, вона змушена чекати в загальній черзі повідомлень. Затримка у відповіді знижує емоційний імпульс до покупки.

**Втрачені вечірні ліди:** Цільова аудиторія - батьки. Зазвичай вони планують дитячі свята ввечері, після роботи. Менеджери в цей час не працюють, тому бізнес втрачає "гарячі" запити.

> **Важливо!**
> 
> Ці проблеми потрібно обов'язково додатково з'ясувати та підтвердити на брифінгу з клієнтом.

## Ядро аудиторії

**Сім’ї, які хочуть відсвяткувати день народження дитини, переважно віком 6–11 років.**

### Чому саме вони

1. Для цієї категорії розроблено велику кількість послуг.
2. Послуга святкування дня народження винесена в окрему вкладку на сайті та доступна в кожному парку.

   ![Хедер EPILAND](./assets/epiland-header.png)

3. Атракціони добре підходять для дітей віком 6–11 років.
4. Для цієї категорії є спеціальні пропозиції:

   - [Пакетний тариф "День народження" - Чабани](https://chabany.epiland.com/paketniy-taryf-den-narodzhennya)
   - [Святковий бонус до дня народження - Обухів](https://obukhiv.epiland.com/svyatkoviy-bonus-do-dnya-narodzhennya)
   - [Пакетний тариф "День народження" - Київ](https://kyiv.epiland.com/paketniy-taryf-den-narodzhennya)
   - [Game Combo для іменинника та друзів - Обухів](https://obukhiv.epiland.com/game-combo-za-super-tsinoyu-dlya-imenynyka-ta-druziv)

### Що їх турбує

1. Скільки це коштує?
2. Чим будуть зайняті діти?
3. Які є варіанти з їжею?
4. Чи зможуть дорослі відпочити під час свята?
5. Наскільки це безпечно?

### Що вони хочуть

1. Легко організувати свято.
2. Забезпечити яскраві емоції для дітей.
3. Отримати зрозумілу ціну та вигідні пропозиції.

## Бізнес-фічі

Після поверхневого аналізу клієнтів можна сформувати базову бізнес-логіку чат-бота.

Чат-бот має бути першим кроком взаємодії з клієнтом. Його задача - частково замінити менеджера, відсіяти базові запити та плавно підвести користувача до етапу покупки.

1. **Організувати дня народження**
   Основний сценарій для ядра аудиторії.

2. **Ціни та послуги**
   Закриває базове питання користувача: "**Скільки це коштує?**"

3. **Атракціони**
   Потрібно для користувачів, які хочуть ознайомитися з доступними розвагами.

4. **Контакти та графік роботи**
   Базова інформація для швидкого зв’язку.

5. **ШІ Помічник**
   > Асистент 100% має бути, треба визначитися з формою. Усі повідомлення "не по формату" відправляються асистенту. Коли користувач тисне на кнопку, вона відправляє заготовлене повідомлення як тригер певної дії. Якщо ж повідомлення написане не за форматом — його обробляє асистент, дозволяючи отримати реальний чат, який зможе проконсультувати без втручання людини. (c)

   **Бізнес-ефект від ШІ:** Таким чином бот закриває більшість потреб клієнтів і ефективно розвантажує менеджера, запобігаючи втраті лідів із нестандартними запитами.

   (Про таку можливість бота обов'язково треба розказати на початку діалогу, бо такий функціонал не виглядає тривіальним за наявності кнопок)
